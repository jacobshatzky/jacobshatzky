Youth Digital Gateway - Java

About YDG

Youth in Canada are more connected than ever before — they grew up on the internet. When asked about their experience with Government of Canada websites, words like "confusing", "difficult to navigate" and "not user-friendly" come up.

The Youth Digital Gateway is designed to help youth reach their potential through, Getting a job or gaining experience, Exploring careers and building skills, and Planning education

Users are able to: Search for a job on a Job Bank, Find a summer job, Explore career options based on employment trends, Explore your career options by degree of diploma, Gain experience through Youth Employment and Skills Programs, and Volunteer or serve and build your skills.

This is all possible in our centralized Youth Digital Gateway!


Implementation

Use of Spring Tool Suite with Java and Maven for the back-end, Javascript and React in the front-end. Azure Devops to host and manage the website

I worked on many layers of the API. The entity model, which represents a table in a relational database, we implemented a SQL script for each API to generate the tables, and the fields in each one. Followed by the repository, that outlines the all the logical read and write operations for a specific entity. The service layer for the client to interact with some functionality of the application. The controllers that controls the flow of the data and process incoming requests then returning the data that is requested.

I started creating entity models for the database, these consisted of all the fields for each table, along with getters and setters, and methods for equals, hash code, and string. I also worked on the services and controllers in the API, this is how the front end would retrieve data from the back end. I worked on every layer of the API, the mappers/assemblers, the controllers, services, repositories, Dto’s, and the entities. Furthermore, every part of each API needs to be tested to ensure everything is working the way we intend. Generally you want to have 80% code coverage from unit tests. I worked on the unit tests for each API, this means each entity, controller, service, and model needs to have unit tests, for each field and method which can quickly add up. I ended up writing approximately 400 unit tests for each resource and program API, as well as 800 for the opportunity-finder API. Most of these tests were standard checking to ensure data was retrieved as expected, as well as some checking the endpoints and making sure the controllers were working. All this testing provides the coverage we need while more importantly, finding bugs.

Machine Learning POC - Python

Proof of Concept with Machine Learning. At CRA, there are large and complex IT systems that generate different types of logs including security logs (generated by IT applications, databases, networks, infrastructure), audit logs, lower state logs, information logs, debug logs, amongst others. For instance, IT security logs are created when any of the following activities are requested or performed by the system or an application; authentication, authorization, system or application changes, availability issues, resource issues, vulnerabilities, etc. The generated logs are monitored and managed by various IT areas at the CRA, they are used for analysis, identification of incidents, and reporting. These type of logs could be monitored and analyzed either manually or automatically. We used Machine Learning to automate this process and leverage unique AI models to read and analyze large amounts of data and report issues/anomalies. Using machine learning to build an automated log anomaly detector will allow us to detect anomalies in real time

We used a combination of unsupervised models, for instance Isolation Forest, Local Outlier Factor, One-Class Support Vector Machine, and more. After this I started looking into more advanced methods, for instance with Neural Networks.

Euchre Card Game - C

The following functions must be implemented: isValidRank(char c), isValidSuit(char c), and getTrump(). All four users will be repeatedly prompted to enter a card until a valid one has been entered. I had to determine which card/player wins the round. For instance, a card of type trump will always beat a card that is not of type trump. If two cards have the same suit, the highest rank is used

Maze - Java

The program will trace through the maze and count the perimiter of a series of connected pixels. • If the position to the left, right, above, or below has ink then the program will perfrom a recursive call on the next pixel. • The program will start on some arbitrary pixel, and determine how many pixels (connected to the starting position) have ink. • The program checks if there is a wall to the left, right, above, or below, if there is it will not call the function in that direction. • After perfoming basic edge cases the program will determine if there is ink on the pixels around it and make a recursive call for instance making a call to the left, computePixelPerimeters(row,column-1);. • The program will continue to make recursive calls until there are no more pixels left to visit, this is done by assigning a “hasVisited” attribute to every pixel and setting it to true after it has been counted. • The program will output the perimiter of the series of connected pixels in the grid.
